## 2. 화면 위치

2. 동적 경로 (SEO) : `/calculator/stock/avg-price/[slug]` (종목별 전용 페이지)

1. **Breadcrumb Schema**: `홈 > 계산기 홈 > 주식 계산기 > 평단가 계산기` 구조의 JSON-LD 적용
2. **동적 메타데이터**: `[slug]`(종목명)에 따라 Title, Description, Keywords 자동 생성

   - `PWAInstallProvider`를 통한 설치 가능 상태 감지

5. JIKO 플랫폼 홈(`jiko.kr`) 허브 이동 버튼 (Header 및 Footer 영역에 단축 위치)

## 4. 화면 기능
1. 입력 데이터
ㄴ 리스트 형태 표
ㄴㄴ 차수 (기본 3차까지 제공, 최대 10차까지 추가 가능)
ㄴㄴ 매수가(원) : 숫자 입력, 최대 9자리 제한
ㄴㄴ 수량(주) : 숫자 입력, 최대 9자리 제한
ㄴㄴ 매수금액(원) : (자동 계산) 단가 * 수량
ㄴㄴ 삭제 : 해당 차수 삭제 기능 (최소 1개 차수는 유지됨)
ㄴㄴ 합계 자동 계산 :  전체 수량 합산, 전체 매수금액 합산
ㄴ 별도 입력 필드
ㄴ 현재가(선택) : 숫자 입력, 최대 9자리 제한

2. 출력 데이터
ㄴ 계산 결과 : 평균 단가, 합계 수량, 매수 합계 금액, (현재가 입력 시) 현재 평가금액, 수익금, 수익률
ㄴ (현재가 입력 및 물려있을 경우) 본전되려면 추가매수 해야할 수량 계산 출력

ㄴ 초기화 : 모든 입력 데이터 조건 초기화 및 3차 기본 셋팅, 흔들림 애니메이션 효과 적용
ㄴ 계산하기 : 계산 결과, 금액 비교 하단 출력
ㄴ 결과 복사 : 평균 단가, 수량, 금액 및 수익률 등의 결과를 텍스트 클립보드에 복사

ㄴ 모바일/PC 반응형 테이블 적용: 모바일에서는 각 컬럼이 최소 너비(90px)를 유지하여 스크롤 발생, PC에서는 여유있는 비율(25%)로 자동 확장.

## 5. 계산 및 비즈니스 로직
1. **차수별 매수 금액** = 매수 단가 × 수량
2. **총 매수 수량** = 각 차수 수량의 합 (빈 값이나 0은 0으로 합산)
3. **총 매수 금액** = 각 차수별 매수 금액의 합
4. **평균 단가** = 총 매수 금액 ÷ 총 매수 수량
5. **현재가 수익금** = (현재가 × 총 매수 수량) - 총 매수 금액
6. **현재가 수익률** = (현재가 수익금 ÷ 총 매수 금액) × 100 (소수점 둘째자리까지 표기)
7. **본전 추가매수 수량 도출 로직** = `Math.ceil(총 매수 금액 / 현재가 - 총 매수 수량)`
8. **수익률 등급 로직**:
   - 30% 이상: 주식 고수 🏆 / 20% 이상: 주식 달인 👑 / 10% 이상: 손절 불가자 ✨
   - 5% 이상: 주식 초보 😅 / 0% 이상: 주식 호구 🐥 / -10% 이상: 묻지마 투자자 🤷 / 그 미만: 야수의 심장 🔥

## 6. 예외 처리
1. 입력값 검증
ㄴ 매수가, 현재가, 수량은 숫자만 입력 가능
ㄴ 매수가, 현재가, 수량은 음수 입력 불가능
ㄴ 매수가, 현재가, 수량은 9자리까지만 입력 가능
2. 계산 결과 검증
ㄴ 매수 금액, 현재 금액, 수익금, 수익률은 숫자만 출력
ㄴ 매수 금액, 현재 금액, 수익금, 수익률은 음수 출력 불가능
ㄴ 매수 금액, 현재 금액, 수익금, 수익률은 9자리까지만 출력

- 매수 단가, 수량, 현재가 모두 문자열/음수 입력 문자 방지 (숫자만 허용, 정규식으로 필터링)
- **최대 입력 길이 제한**: 매수가, 수량, 현재가 모두 **9자리(10억 미만)** 까지만 입력 가능하도록 차단.
- 빈 값이 포함된 경우 필터링하거나 0으로 계산에서 제외.
- 총 수량이 0일 경우, 평균 단가를 Number.NaN 또는 무한대로 처리하지 않고 `0`으로 일괄 처리.
