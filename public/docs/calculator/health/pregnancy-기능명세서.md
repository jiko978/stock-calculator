# pregnancy-기능명세서.md

## 1. 개요
현재 임신 주수 및 출산 예정일을 계산하는 메뉴의 기능 명세서입니다.

## 2. 화면 위치
1. 경로 : `/calculator/health/pregnancy`
2. 동적 경로 (SEO) : `/calculator/health/pregnancy/[slug]`
3. 파일
ㄴ 서버 : `app/calculator/health/pregnancy/page.tsx`
ㄴ 동적 서버 : `app/calculator/health/pregnancy/[slug]/page.tsx`
ㄴ 클라이언트 : `app/calculator/health/pregnancy/Pregnancy.tsx`

## 3. 계층 구조 및 SEO (Common Architecture)
1. **Breadcrumb Schema**: `홈 > 계산기 홈 > 건강 계산기 > 임신주수 계산기` 구조의 JSON-LD 적용
2. **동적 메타데이터**: `[slug]`에 따라 Title, Description, Keywords 자동 생성
3. **PWA 연동**: 
   - `PWAInstallProvider`를 통한 설치 가능 상태 감지
   - `InstallBanner` (페이지 하단), `Footer` (앱 설치 버튼) 연동
4. 뒤로가기 버튼 / 공유 버튼 (NavBar 컴포넌트 필수 사용)
5. JIKO 플랫폼 홈(`jiko.kr`) 허브 이동 버튼 (Header 및 Footer 영역에 단축 위치)

## 4. 화면 기능
1. 입력 데이터
ㄴ 계산 기준 (탭 또는 셀렉트박스로 3가지 옵션 중 하나 선택)
   - A. 마지막 생리 시작일 (기본값)
   - B. 초음파상 임신 주수 (예: 오늘이 N주 N일째)
   - C. 출산 예정일 (이미 예정일을 알고 있을 경우 역산)
ㄴ 선택한 기준에 따른 날짜(YYYY-MM-DD) 또는 주수 입력

2. 출력 데이터
ㄴ 출산 예정일 (D-Day 포함)
ㄴ 현재 임신 주수 (N주 N일차)
ㄴ 임신 진행률 프로그레스 바(Progress Bar) (0% ~ 100%)
ㄴ 현재 1분기/2분기/3분기 (Trimester) 구분 및 간단한 상태 코멘트
ㄴ 결과 복사 기능 (텍스트 클립보드 복사)

3. 버튼 기능
ㄴ 초기화 : 입력 데이터 당일 및 기본 탭으로 초기화
ㄴ 계산하기 : 계산 결과 출력
ㄴ 결과 복사 : 결과 텍스트 복사

4. UI/UX 기능
ㄴ 직관적인 날짜 선택기(Date Picker) 제공
ㄴ 진행률을 직관적으로 보여주는 둥근 형태의 프로그레스 바 애니메이션 적용
ㄴ 초기화 버튼 클릭 시 흔들림 애니메이션 (shake) 효과 적용

5. 하단 정보 영역 (SEO 및 사용자 가이드 - page.tsx)
ㄴ 검색 엔진 최적화(SEO)를 위한 JSON-LD 라우팅 보강
ㄴ UI 디자인 : 본체와 동일한 카드형 레이아웃 및 다크모드 대응 적용
ㄴ 구성요소
ㄴㄴ 소개 : 임신 주수 및 출산 예정일 계산기 목적 안내
ㄴㄴ 사용 방법 : 상황에 맞게 생리일, 주수, 예정일을 기준으로 계산하는 방법 안내
ㄴㄴ 계산 예시 : 평균 280일 기준의 임신 기간 계산 예시
ㄴㄴ FAQ : 네겔레(Naegele) 공식 등 출산 예정일 계산법 안내 (**공통 `FAQ` 컴포넌트 적용**)
ㄴㄴ 건강 계산기 더 보기 : 다른 건강 계산기 메뉴 바로가기 (**공통 `HealthMoreCalculators` 컴포넌트 적용**)

## 5. 계산 로직
1. 마지막 생리 시작일 기준 (네겔레 공식 기반 통상 280일):
   - 출산 예정일: `마지막 생리 시작일 + 280일(40주)`
   - 현재 임신 주수: `(오늘 날짜 - 생리 시작일) / 7` 의 몫(주)과 나머지(일)
2. 초음파상 주수 기준:
   - 기준일(선택한 날짜)에서 입력된 주수(N주 N일)를 역방향으로 빼서 실제 '임신 1일차'를 도출
   - 도출된 임신 1일차 기준으로 남은 기간 및 출산 예정일 계산
3. 출산 예정일 기준:
   - 출산 예정일에서 280일을 역방향으로 빼서 임신 1일차를 역산
   - 1일차부터 오늘까지 경과한 날짜를 바탕으로 현재 주수 계산
*(모든 계산 결과는 단순 달력 계산이므로 개인별 차이가 날 수 있으며 병원 진료가 가장 정확하다는 문구 포함)*

## 6. 예외 처리
1. 미래의 날짜를 생리 시작일로 입력하는 등 논리적으로 맞지 않는 시점 입력 시 경고
2. 입력된 예정일이 이미 과거 날짜인 경우, 임신 기간(280일)이 종료되었다는 문구 등 예외 처리
3. 단순 정보 제공용으로 실제 초음파 결과와 며칠 차이가 날 수 있다는 의학적 면책 조항 노출

## 7. 스타일(디자인) 참고
- 프로젝트 공통 CSS(tailwind) 및 기타 계산기 디자인 패턴 준수.
- 임신/태아 성장을 은유적으로 나타내는 D-day 표시 및 프로그레스 바(Progress Bar)의 시각적 요소 중요.
