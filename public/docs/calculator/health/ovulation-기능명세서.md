# ovulation-기능명세서.md

## 1. 개요
배란일 및 가임기, 다음 생리 예정일을 계산하는 메뉴의 기능 명세서입니다.

## 2. 화면 위치
1. 경로 : `/calculator/health/ovulation`
2. 동적 경로 (SEO) : `/calculator/health/ovulation/[slug]`
3. 파일
ㄴ 서버 : `app/calculator/health/ovulation/page.tsx`
ㄴ 동적 서버 : `app/calculator/health/ovulation/[slug]/page.tsx`
ㄴ 클라이언트 : `app/calculator/health/ovulation/Ovulation.tsx`

## 3. 계층 구조 및 SEO (Common Architecture)
1. **Breadcrumb Schema**: `홈 > 계산기 홈 > 건강 계산기 > 배란일 계산기` 구조의 JSON-LD 적용
2. **동적 메타데이터**: `[slug]`에 따라 Title, Description, Keywords 자동 생성
3. **PWA 연동**: 
   - `PWAInstallProvider`를 통한 설치 가능 상태 감지
   - `InstallBanner` (페이지 하단), `Footer` (앱 설치 버튼) 연동
4. 뒤로가기 버튼 / 공유 버튼 (NavBar 컴포넌트 필수 사용)
5. JIKO 플랫폼 홈(`jiko.kr`) 허브 이동 버튼 (Header 및 Footer 영역에 단축 위치)

## 4. 화면 기능
1. 입력 데이터
ㄴ 마지막 생리 시작일 (날짜 선택: YYYY-MM-DD)
ㄴ 평균 생리 주기 (기본값: 28일, 범위: 20~45일 숫자 혹은 셀렉트 박스 선택)

2. 출력 데이터
ㄴ 다음 생리 예정일
ㄴ 배란 예정일
ㄴ 가임기 시작일 ~ 종료일
ㄴ 결과를 직관적으로 보여주는 시각화 자료 (예: 미니 달력 형태나 컬러 라인)
ㄴ 결과 복사 기능 (텍스트 클립보드 복사)

3. 버튼 기능
ㄴ 초기화 : 입력 데이터 당일 및 기본값으로 초기화
ㄴ 계산하기 : 계산 결과 출력
ㄴ 결과 복사 : 결과 텍스트 복사

4. UI/UX 기능
ㄴ 날짜 입력을 편하게 할 수 있는 네이티브 방식의 Date Picker 제공
ㄴ 초기화 버튼 클릭 시 흔들림 애니메이션 (shake) 효과 적용

5. 하단 정보 영역 (SEO 및 사용자 가이드 - page.tsx)
ㄴ 검색 엔진 최적화(SEO)를 위한 JSON-LD 라우팅 보강
ㄴㄴ `SoftwareApplication` 및 `FAQPage` 외 추가 범용 `schema` 스크립트 작성 및 로드
ㄴ UI 디자인 : 본체와 동일한 카드형 레이아웃 및 다크모드 대응 적용
ㄴ 구성요소
ㄴㄴ 소개 : 배란일, 가임기, 다음 생리 예정일 계산기 목적 안내
ㄴㄴ 사용 방법 : 마지막 생리 시작일과 주기를 입력하여 날짜를 확인하는 가이드
ㄴㄴ 계산 예시 : 생리 주기 28일 기준 배란일 도출 예제
ㄴㄴ FAQ : 배란일 계산 공식, 가임기 계산 방법 등 안내 (**공통 `FAQ` 컴포넌트 적용**)
ㄴㄴ 건강 계산기 더 보기 : 다른 건강 계산기 메뉴 바로가기 (**공통 `HealthMoreCalculators` 컴포넌트 적용**)

## 5. 계산 로직
1. 다음 생리 예정일: `마지막 생리 시작일 + 평균 생리 주기`
2. 배란 예정일: `다음 생리 예정일 - 14일`
3. 가임기: `배란 예정일 - 5일` ~ `배란 예정일 + 2일` (또는 배란일 전후 3~5일 기준)
*(단, 생리 주기가 불규칙한 경우 계산 결과가 부정확할 수 있다는 안내 문구 필수 포함)*

## 6. 예외 처리
1. 날짜가 올바르게 선택되지 않은 경우 혹은 미래의 날짜인 경우 예외 처리
2. 생리 주기가 20일 미만이거나 40일 초과 등 비정상적인 범위를 입력할 경우 경고 문구 표시
3. 계산 결과는 전문적인 의학적 진단/피임 수단을 대체할 수 없다는 면책 조항(Disclaimer) 노출
